Our Philosophy

VisionPipe3D emerged from a simple idea: your hands should control your digital world. We move fast, ship often, and maintain quality through disciplined branching.

One mission. Make gesture control accessible to every developer.

Branch Types

main
Production-ready code. Every commit is deployable, tested, and documented. Direct pushes are not allowed.
develop
Integration layer where new code meets existing functionality. Run the full test suite before merging to main.
feature/*
Building something new? Create a feature branch. Name it clearly, work iteratively, keep commits focused.
fix/*
Found a bug? Isolate the fix. Reference the issue, test thoroughly, submit for review.

Quick Commands

# Start a new feature
git checkout -b feature/palm-rotation-tracking develop

# Start a bug fix
git checkout -b fix/camera-permission-handler develop

# Update your branch
git pull origin develop

# Push changes
git push -u origin your-branch

Pull Request Process

1

Create

Branch from develop with a clear, descriptive name.

2

Build

Implement your feature or fix with focused commits.

3

Test

Verify locally with webcam enabled for hand tracking features.

4

Review

Open a pull request. Reviews happen within 24 hours.

5

Merge

After approval, your code joins the project.
